# IP Assignment and Work-for-Hire Clause Drafter
## Standard Clauses

### Client-Favorable (Full Transfer)
```
All work product created under this agreement ("Deliverables") shall be
considered work made for hire. Upon full payment, all intellectual property
rights in the Deliverables shall transfer to Client. Contractor retains no
rights to use, reproduce, or distribute the Deliverables.
```

### Contractor-Favorable (License Model)
```
Contractor grants Client a perpetual, non-exclusive, royalty-free license
to use the Deliverables for Client's business purposes. Contractor retains
ownership of all intellectual property and may reuse code, methodologies,
and frameworks in other projects. Pre-existing IP remains Contractor's property.
```

### Balanced (Recommended Default)
```
Upon full and final payment, Client receives all rights to the custom
Deliverables created specifically for this project. Contractor retains
ownership of: (a) pre-existing tools, libraries, and frameworks; (b)
general-purpose code and methodologies; (c) the right to use anonymized
project details in portfolio. Contractor grants Client a perpetual license
to any pre-existing components incorporated into the Deliverables.
```

## Key Considerations
- Pre-existing IP must be explicitly carved out
- Open-source components: client gets license, not ownership
- Portfolio rights: negotiate separately
- IP transfer conditional on full payment (always)
