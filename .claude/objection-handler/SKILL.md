# Objection Handler
## Common Objections and Response Frameworks

### Price Objection
```
ACKNOWLEDGE: "I understand budget is a key consideration."
REFRAME: Focus on ROI, not cost. "The question isn't what it costs — it's what it costs you NOT to solve this."
STRATEGIES:
A) Value stack: Break down deliverables and show value per component
B) Phased approach: Offer MVP first, expand later
C) Comparison: "Hiring in-house for this would cost [X]/month"
```

### Timing Objection
```
STRATEGIES:
A) Create urgency: "Every month without [solution], you're losing [X]"
B) Easy start: "We can begin with a small discovery phase while you finalize budget"
C) Future-lock: "If we start the SOW now, we can lock in current pricing"
```

### Trust/Risk Objection
```
STRATEGIES:
A) Social proof: Share relevant case study
B) Risk reversal: Offer satisfaction milestone before full commitment
C) Pilot project: Suggest a small paid pilot to demonstrate capability
```

### Competitor Objection
```
STRATEGIES:
A) Differentiate: "Unlike [competitor], we [unique value]"
B) Acknowledge: "They're solid for [X]. Where we differ is [Y]"
C) Total cost: Compare total cost of ownership, not sticker price
```
