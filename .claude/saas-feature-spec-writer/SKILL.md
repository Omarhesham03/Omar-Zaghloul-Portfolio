# SaaS Feature Spec Writer
## Feature Spec Template
```
FEATURE SPECIFICATION
=====================
Feature: [Name]
Author: [Name]
Date: [Date]
Status: [Draft / Review / Approved]
Priority: [P1-P4]

PROBLEM STATEMENT:
[What user pain this solves — 2-3 sentences]

USER STORIES:
- As a [role], I want to [action] so that [benefit]
- As a [role], I want to [action] so that [benefit]

ACCEPTANCE CRITERIA:
Given [precondition]
When [action]
Then [expected result]

EDGE CASES:
1. [Edge case]: [Expected behavior]
2. [Edge case]: [Expected behavior]

UI/UX REQUIREMENTS:
- [Wireframe description or reference]
- [Interaction behavior]

TECHNICAL CONSIDERATIONS:
- API changes required: [Yes/No — details]
- Database changes: [Yes/No — details]
- Third-party dependencies: [List]
- Performance impact: [Assessment]
- Security considerations: [Assessment]

OUT OF SCOPE:
- [What this feature explicitly does NOT do]

SUCCESS METRICS:
- [Metric 1]: [Target]
- [Metric 2]: [Target]
```
