# Changelog and Release Notes Generator
## Changelog Format (Keep a Changelog)
```
# Changelog

## [Version] — [Date]
### Added
- [New feature description]
### Changed
- [Modified behavior description]
### Fixed
- [Bug fix description]
### Removed
- [Removed feature description]
### Security
- [Security fix description]
### Deprecated
- [Feature marked for future removal]
```

## User-Facing Release Notes
```
## What's New in [Version]

🆕 **[Feature Name]**
[1-2 sentence user-friendly description of what it does and why they care]

🔧 **Improvements**
- [Improvement in user-friendly language]

🐛 **Bug Fixes**
- [Fix in user-friendly language]

📢 **Coming Soon**
- [Teaser for next release]
```

## Rules
- Changelogs are for developers (technical, complete)
- Release notes are for users (benefits-focused, curated)
- Group by category, not by commit
- Lead with the most impactful change
- Link to docs for complex features
