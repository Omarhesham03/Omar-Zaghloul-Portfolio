# Personal Knowledge Base Manager
## Structure
```
knowledge-base/
├── projects/
│   ├── [project-name]/
│   │   ├── notes.md
│   │   ├── decisions.md
│   │   └── learnings.md
├── research/
│   ├── [topic]/
│   │   ├── papers.md
│   │   ├── key-findings.md
│   │   └── references.bib
├── tools/
│   ├── [tool-name].md (setup, tips, gotchas)
├── templates/
│   ├── [template-type].md
├── contacts/
│   ├── professional.md
├── learnings/
│   ├── [year]-[month].md (monthly learnings log)
└── inbox/
    └── unsorted.md (quick capture, process weekly)
```

## Note Template
```
# [Topic]
Date: [Date]
Source: [Where this came from]
Tags: [tag1, tag2]

## Key Points
- [Point 1]
- [Point 2]

## My Take
[Your interpretation, how it connects to your work]

## Action Items
- [ ] [What to do with this information]
```

## Weekly Processing
1. Move items from inbox/ to appropriate folders
2. Tag and cross-reference new entries
3. Review and update active project notes
4. Archive completed project notes
